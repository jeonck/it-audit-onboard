# 기술적 점검 방법 및 결과 기록 양식

살펴본 4가지 주요 사업 유형에 대해 감리 현장에서 실제 사용하는 **기술적 점검 방법(How-to)**과 결과 기록 양식(Template) 예시를 정리합니다.

### 1. 클라우드 네이티브: 기술 점검 및 양식

클라우드 환경은 설정(Configuration)이 곧 인프라입니다. 따라서 코드화된 인프라(IaC) 및 설정값 검증이 핵심입니다.

**기술적 점검 방법:**
- **Auto-scaling 검증**: 부하 발생 도구(JMeter 등)를 사용하여 설정된 임계치(CPU 70% 등)에서 Pod가 자동으로 늘어나는지 모니터링 툴(Grafana, Lens 등)로 확인.
- **장애 격리 검증**: 특정 마이크로서비스를 강제로 중단시켰을 때, API Gateway에서 해당 서비스로의 요청을 차단(Circuit Breaking)하고 타 서비스는 정상 작동하는지 확인.

**점검 양식(예시):**

| 점검항목 | 테스트 시나리오 | 기대 결과 | 실제 결과 | 판정 |
|----------|----------------|-----------|-----------|------|
| 자동 확장성 | CPU 부하 80% 이상 지속 발생 | Replica Set이 2개에서 5개로 증가 | 5개로 정상 증설 확인 | 적합 |
| 장애 전파방지 | 결제 서비스 강제 종료 | 메인 페이지는 정상 호출됨 | 타 서비스 영향 없음 확인 | 적합 |

### 2. 인공지능(AI): 기술 점검 및 양식

AI 감리는 모델의 **성능 지표(Metric)**와 데이터 흐름을 기술적으로 검증해야 합니다.

**기술적 점검 방법:**
- **성능 평가서 검증**: 수행사가 제출한 Test Set 기반의 혼동 행렬(Confusion Matrix)을 재검증. (Precision, Recall, F1-Score 등 계산)
- **할루시네이션 테스트**: 시스템 프롬프트(System Prompt) 외의 질문이나 유도 심문에 모델이 거부하거나 정확한 근거(RAG)를 대는지 직접 질의응답(QA) 수행.

**점검 양식(예시):**

| 평가 지표 | 목표치 | 측정 결과 | 검증 도구 | 비고 |
|-----------|--------|-----------|-----------|------|
| 정확도(Accuracy) | 92.0% 이상 | 94.5% | Python Scikit-learn | 테스트 데이터셋 1만 건 |
| 응답 속도(Latency) | 2초 이내 | 1.8초 | API 호출 로그 분석 | GPU 서버 점유율 60% 시 |

### 3. DB 통합 및 이관: 기술 점검 및 양식

데이터 이관은 샘플링이 아닌 전수 검증 원칙을 기술적으로 확인해야 합니다.

**기술적 점검 방법:**
- **데이터 무결성 검증**: SQL 쿼리를 통해 AS-IS와 TO-BE의 COUNT(*), SUM(금액), AVG(값) 등을 비교하여 데이터 유실 여부 확인.
- **표준 준수 자동 검증**: 공공 데이터 품질관리 도구(DQC) 등을 활용하여 표준 단어/용어 사전과 실제 스키마의 일치도를 자동 점검.

**점검 양식(예시):**

| 테이블명 | AS-IS 건수 | TO-BE 건수 | 오차 건수 | 주요 정합성 검증 쿼리 결과 |
|----------|------------|------------|-----------|---------------------------|
| 회원정보 | 1,200,500 | 1,200,500 | 0 | SELECT SUM(mileage) FROM users 일치 |
| 결제내역 | 5,430,210 | 5,430,210 | 0 | SELECT COUNT(*) FROM payment 일치 |

### 4. 민간 클라우드(SaaS): 기술 점검 및 양식

민간 서비스는 보안 설정과 접속 로그의 가시성을 확보하는 것이 중요합니다.

**기술적 점검 방법:**
- **인증 보안성 테스트**: 관리자 페이지 접속 시 2차 인증(MFA) 요구 여부 확인 및 비밀번호 복잡도 정책 적용 확인.
- **데이터 파기 기술 확인**: 암호화된 데이터의 복구 불가능한 파기(Key Shredding) 알고리즘이 설계서 및 실제 가동 환경에 적용되었는지 확인.

**점검 양식(예시):**

| 보안 요구사항 | 설계 반영 여부 | 실제 구현 확인 | 확인 방법 | 판정 |
|---------------|----------------|----------------|-----------|------|
| MFA 적용 | Y | OTP/문자 인증 연동 | 실제 관리자 로그인 테스트 | 적합 |
| 비인가 차단 | Y | 지정된 IP 외 접근 차단 | 외부망 접속 시도(Access Denied) | 적합 |
