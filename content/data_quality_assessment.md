# 데이터 품질 진단

## 1. 데이터 품질 진단 기준

| 품질기준 | 세부 품질기준 | 품질기준 설명 | 활용 예시 |
|---------|-------------|-------------|----------|
| 완전성 (Completeness) | 개별 완전성 | 필수 컬럼에 값의 누락이 없는지 여부 | 고객의 아이디는 NULL일 수 없다. |
| | 조건 완전성 | 조건에 따라 컬럼 값이 항상 존재하는지 여부 | 기업 고객 레코드의 경우는 사업자 등록번호가 NULL일 수 없다. |
| 유효성 (Validity) | 범위 유효성 | 주어진 범위 내에 컬럼 값이 존재하는지 여부 | 여부 도메인은 'Y', 'N', '0', '1' 중 하나의 값을 가진다. |
| | 날짜 유효성 | 날짜 유형의 컬럼이 유효 날짜 값을 가지는지 여부 | 99991231, 20080231은 유효하지 않은 값이다. |
| | 형식 유효성 | 컬럼이 정해진 형식과 일치하는 값을 가지는지 여부 | 주민번호 형식은 '999999-9999999'의 형식이어야 한다. |
| 일관성 (Consistency) | 참조 무결성 | 테이블 간의 컬럼 값이 참조 관계에 있을 경우, 참조 무결성을 유지하고 있는지 여부 | 대출상세 내역의 대출원장 번호는 대출 원장에 존재해야 한다. |
| | 코드 무결성 | 공통 코드에 정의한 코드값 만을 사용하는지 여부 | 공통코드에서 정의한 '01', '02' 이외의 값을 사용해서는 안 된다. |
| | 값 일관성 | 동일 개체의 동일 컬럼에 대해 다른 값이 존재하지 않는지 여부 | 동일 고객 ID를 가진 레코드의 이름이 '홍길동'과 '홍길서'로 다르게 기록되어서는 안 된다. |
| | 관계 일관성 | 두 개 이상의 컬럼 값 간에 논리적인 모순이 없는지 여부 | 입사일이 퇴사일보다 늦을 수 없으며, 나이가 0세 이하일 수 없다. |
| 정확성 (Accuracy) | 실제 값 일치성 | 데이터 값이 현실의 실제 값과 일치하는지 여부 | 고객의 실제 성별이 남성일 경우, 성별 컬럼 값도 '남'이어야 한다. |
| | 오류 없음 | 데이터 입력, 변환, 저장 과정에서 오류가 없는지 여부 | 원장의 금액 총합이 상세 내역의 금액 총합과 일치해야 한다. |
| 유일성 (Uniqueness) | 식별자 유일성 | 주요 식별자 컬럼의 값이 유일한지 여부 | 고객 ID는 모든 레코드에서 중복되지 않고 하나만 존재해야 한다. |
| | 코드 유일성 | 중복된 레코드가 존재하지 않는지 여부 | 동일한 내용을 가진 고객 정보 레코드가 두 번 이상 존재해서는 안 된다. |
| 접근성 (Accessibility) | 사용의 편리성 | 데이터 사용 및 접근이 편리하도록 관리되는지 여부 | 데이터 항목명, 정의, 사용 방법 등에 대한 메타데이터를 제공해야 한다. |
| | 접근 환경의 적절성 | 데이터를 접근하고 활용하는 환경이 적절하게 제공되는지 여부 | 데이터 조회 및 다운로드 기능이 안정적으로 제공되어야 한다. |

## 2. 데이터 품질 진단 방법의 유형

| 구분 | 내용 |
|-----|------|
| 프로파일링 (Profiling) | **값 진단**: 데이터 값의 유효성, 정확성 등 데이터 값 자체 오류를 분석하는 방법<br>- 컬럼분석, 날짜분석, 패턴분석, 코드분석 등을 통해 데이터 값의 정확성을 중심으로 진단 |
| | **구조 진단**: 논리적 데이터 구조의 오류로 인한 일관성, 정합성 등을 확보하지 못하는 결함을 분석하고 진단하는 방법<br>- 표준화 수준(코드, 도메인 등), 테이블 구조, 정규화 수준, 컬럼 및 관계 정의 등 데이터의 구조적 결함 측정 |
| 업무규칙 진단 (Business Rule Diagnosis) | 법, 규정에 정의된 업무기준(산출식)에 근거하여 데이터가 관리되고 있는지를 진단하는 방법<br>- 업무규칙(BR; Business Rule)을 준수하고 있는지에 관한 측정 스크립트(SQL등)를 실행하여 오류 값을 추출 |
| 체크리스트 (Checklist) | 전반적인 데이터 품질관리 수준과 지표별 데이터 품질 수준을 체크리스트(설문 또는 인터뷰)를 통해 진단하는 방법 |
| 비정형 실측 (Unstructured Data Measurement) | 문서, 이미지, 동영상 등 정형화되어 있지 않은 정보를 사람이 직접 확인(실측)하여 오류 여부를 진단하는 방법<br>- 별도 도구 없이 직접 정보를 조회하거나 해당 문서를 수기로 확인 등 |

## 3. 진단 및 검증 절차

| 구분 | 내용 |
|-----|------|
| 계획 및 분석 | - 범위선정, 환경설정 및 업무분석<br>- 진단을 위한 툴 준비 및 SQL 작성 |
| 데이터품질 진단 | - 데이터품질 진단<br>- 데이터 값에 대한 실증 점검<br>- 데이터 프로파일링 및 데이터 분석 |
| 결과분석 및 보고서 작성 | - 결과 분석 및 보고서 작성 |

## 4. 진단 방법 및 진단 및 검증 결과

| 진단 방법 | 상세 진단 방법 | 진단 및 검증 결과 |
|----------|--------------|-----------------|
| 데이터 프로파일링 | 데이터 품질 측정 및 개선방향 제시<br>- 품질진단 대상 선정<br>- 데이터 유형별 품질 진단<br>- 개발 환경 DB에 적재된 데이터 값의 품질 진단<br>- 데이터 품질 개선방향 제시 | (1) 참조 |
| 업무규칙 진단 | 데이터 품질 측정 및 개선방향 제시<br>- 진단 대상 선정 및 업무 식별<br>- 업무 규칙 정의 확인<br>- 개발 환경 DB에 적재된 데이터 값의 품질 진단<br>- 데이터 품질 개선방향 제시 | (1) 참조 |
