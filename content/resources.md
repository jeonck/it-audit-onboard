# 참고 자료 및 도구

## 법령 및 표준

### 1. 법령 및 행정규칙 (최우선 준수)
- **전자정부법 / 동법 시행령**: 감리 수행의 법적 근거 및 대상 사업 규정
- **정보시스템 감리기준 (행정안전부 고시)**: 감리 절차, 인력 배치, 대가 산정, 검사항목 등 실무 표준
- **소프트웨어 진흥법**: SW 품질보증 및 과업 변경, 적정 대가 산정 관련 준수

### 2. 보안 및 취약점 가이드라인 (통제 분야)
- **SW 개발 보안 가이드 (행정안전부)**: 이른바 '시큐어 코딩' 가이드. 공공사업 감리 시 필수 점검 항목
- **KISA 보안 가이드라인**: 클라우드 서비스 보안인증(CSAP) 가이드, 개인정보 보호 기술적·관리적 보호조치 기준
- **OWASP Top 10**: 웹 애플리케이션의 10대 보안 취약점 (국제 표준 가이드)
- **ISO/IEC 29147**: 보안 취약점 공개 절차에 관한 국제 표준 (취약점 관리 체계 점검 시 활용)
- **주요 정보통신기반시설 취약점 분석·평가 가이드 (국정원/KISA)**: 서버, 네트워크, DB 등 인프라 보안 점검의 기준

### 3. 품질 및 산출물 표준 (기술 분야)
- **ISO/IEC 25010 (구 ISO/IEC 9126)**: 소프트웨어 품질 모델. 기능 적합성, 성능 효율성, 사용성, 신뢰성 등 품질 특성 점검 기준
- **ISO/IEC 12207**: 소프트웨어 생명주기 프로세스 표준
- **공공기관 웹 사이트 구축·운영 가이드 (행정안전부)**: 웹 표준 준수, 접근성(WA), 호환성 점검 기준

### 4. 인프라 및 기타 (참고 분야)
- **정보통신공사 감리기준 (과학기술정보통신부)**: 정보시스템 구축 중 물리적 네트워크 선로, 구내 통신 설비 등이 포함된 경우에 한해 부분적으로 적용 (SW 감리와는 별개 영역이나 대규모 통합 구축 시 참고)

## 정보시스템 감리 도구

최근 클라우드 네이티브 환경 확산, AI 기반 자동화, 그리고 제로 트러스트 보안 모델의 도입에 따라 정보시스템 감리 현장에서도 새로운 도구들이 필수적으로 활용되고 있습니다. 2025년 최신 동향을 반영하여 보완 및 확장된 감리 도구 목록입니다.

### [요약 표: 최신 정보시스템 감리 도구 체계]

| 분류 | 주요 도구 (Classic) | 최신 트렌드 도구 (Modern) |
|------|-------------------|------------------------|
| 정적분석 | SonarQube, Fortify | Snyk, CodeQL, AI Code Review |
| 보안점검 | Nessus, Burp Suite | Wiz, Prisma Cloud, Trivy |
| 네트워크 | Wireshark, Nmap | Datadog, Faddom (가시성 확보) |
| DB 분석 | TOAD, DBeaver | Cyera (DSPM), Unity Catalog |
| 성능/장애 | JMeter, LoadRunner | k6, Locust, Gremlin |
| 감리 관리 | Excel, 한글 | AuditBoard, Hyperproof (GRC 자동화) |

### 1. 정적 분석 (Static Analysis & AI Review)

기존의 규칙 기반 분석을 넘어, AI가 코드의 맥락을 이해하고 수정 제안까지 제공하는 도구들이 부상하고 있습니다.

- **기존 도구**: SonarQube, Checkmarx, Fortify SCA
- **최신 추가**:
  - **Snyk**: 오픈소스 라이브러리 취약점(SCA)과 컨테이너 보안을 동시에 점검하는 데 특화되어 있어 클라우드 감리 시 필수적입니다.
  - **CodeQL**: GitHub 기반의 세밀한 쿼리 분석으로 복잡한 보안 취약점을 탐색합니다.
  - **CodeRabbit / Qodo (Codium)**: AI 에이전트가 코드 리뷰를 수행하여 인간 감리인이 놓치기 쉬운 논리적 결함과 성능 병목을 지적합니다.

### 2. 보안 및 클라우드 진단 (Security & CSPM)

단순 웹 취약점 점검을 넘어, 클라우드 설정 오류(Misconfiguration) 및 공급망 보안 점검이 중요해졌습니다.

- **기존 도구**: Nessus, OWASP ZAP, Burp Suite
- **최신 추가**:
  - **Wiz / Prisma Cloud**: 클라우드 설정 오류를 잡아내는 CSPM(Cloud Security Posture Management) 도구로, 공공/금융 클라우드 감리에서 비중이 매우 높습니다.
  - **Trivy**: 컨테이너 이미지와 IaC(코드형 인프라) 설정 취약점을 검사하는 경량화된 오픈소스 도구입니다.
  - **Nuclei**: 최신 취약점 템플릿을 빠르게 업데이트하여 대규모 인프라의 취약점을 자동 스캔합니다.

### 3. 네트워크 및 관제 분석 (Network & Observability)

패킷 분석(Wireshark) 중심에서 애플리케이션 간의 상호작용과 전체 흐름을 시각화하는 도구로 진화했습니다.

- **기존 도구**: Wireshark, Nmap, Tcpdump
- **최신 추가**:
  - **Datadog / New Relic**: 인프라 전반의 모니터링 및 로그 분석을 통해 감리 시 성능 장애의 근본 원인을 파악합니다.
  - **Faddom / Dynatrace**: 하이브리드 클라우드 환경에서 보이지 않는 서버 간 연결 관계를 자동으로 맵핑하여 아키텍처 적정성을 진단합니다.

### 4. 데이터베이스 및 데이터 거버넌스 (DB & Data Governance)

단순 SQL 작성을 넘어 개인정보 보호 준수 여부와 데이터 오남용을 감시하는 기능이 중요합니다.

- **기존 도구**: TOAD, SQL Developer, DBeaver
- **최신 추가**:
  - **Cyera**: 데이터 보안 태세 관리(DSPM) 도구로, 민감 정보가 어디에 저장되어 있고 누가 접근하는지 자동 분류합니다.
  - **Databricks Unity Catalog**: 클라우드 데이터 레이크 환경에서 데이터 권한 및 계보(Lineage)를 추적하여 데이터 거버넌스를 점검합니다.

### 5. 성능 및 부하 테스트 (Performance & SRE)

전통적인 부하 테스트에서 서비스 지속성을 확인하는 카오스 엔지니어링으로 영역이 넓어지고 있습니다.

- **기존 도구**: JMeter, LoadRunner
- **최신 추가**:
  - **k6**: Go 언어 기반의 현대적인 부하 테스트 도구로, CI/CD 파이프라인 연동이 쉬워 개발 단계 감리에 자주 활용됩니다.
  - **Locust**: Python 기반으로 대규모 동시 접속자를 시뮬레이션하며 커스터마이징이 용이합니다.
  - **Gremlin**: 시스템에 의도적으로 장애를 주입하여 복구 능력을 확인하는 카오스 엔지니어링 도구입니다.

이러한 도구들은 특히 SaaS 형태로 제공되는 경우가 많아, 최근 감리에서는 도구 자체의 기술적 숙련도뿐만 아니라 클라우드 콘솔 분석 역량이 함께 요구되는 추세입니다.
